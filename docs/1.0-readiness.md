# vibe CLI — 1.0 Readiness

A standing audit of what's ready, what's intentionally deferred, and the historical breaking changes a v0.x → 1.0 user needs to know about. Updated whenever the public surface or the migration story changes.

> **Status as of v0.91:** Public API surface is stable. No `@deprecated` / `@internal` / `@experimental` markers anywhere. Drift checks pass. Ready for a 1.0 cut whenever the team chooses.

## What "1.0" means here

- **Public API freeze**: `@vibeframe/cli`, `@vibeframe/ai-providers`, `@vibeframe/mcp-server` exports become semver-major-protected. Breaking changes require a `2.0`.
- **Command-tree freeze**: every command currently visible in `vibe schema --list` is supported. Removing one is breaking; adding one is minor.
- **Cost-tier system stable**: the four-tier vocabulary (`free` / `low` / `high` / `very-high`) and the agent-side `medium` tier are part of the contract. New tiers only via major.
- **Config schema (`~/.vibeframe/config.yaml`) frozen**: shape additions are minor; field renames are major.

## Public API surface (npm consumers)

### `@vibeframe/cli`

- **Engine**: `Project` class, `generateId()`, `ProjectFile` type
- **Agent**: `startAgent()`, `AgentExecutor`, `ToolRegistry`, `ConversationMemory`, `AgentConfig`, `AgentContext`, `AgentMessage`, `ToolCall`, `ToolResult`, `LLMAdapter`
- **Config**: `loadConfig()`, `saveConfig()`, `isConfigured()`, `VibeConfig`
- **Cost-tier**: `getCostTier()`, `TIER_DESCRIPTION`, `CostTier`
- **Submodule entry points** via `package.json#exports`: `./engine`, `./agent`, `./tools/manifest`, `./tools/define-tool`, `./tools/adapters/{mcp,agent}`, `./pipeline`, plus 11 detailed `./commands/*` paths

### `@vibeframe/ai-providers`

- 14 provider classes (`WhisperProvider`, `GeminiProvider`, `OpenAIProvider`, `ClaudeProvider`, `OllamaProvider`, `ElevenLabsProvider`, `KokoroProvider`, `OpenAIImageProvider`, `RunwayProvider`, `KlingProvider`, `GrokProvider`, `FalProvider`, `ReplicateProvider`)
- Registry: `defineApiKey`, `defineProvider`, `getProvidersFor`, `getProviderEnvVars`, `getCommandKeyMap`, `getSetupProviders`, `getAllApiKeys`, `getAllProviders`, `getKeyFormat`
- Metadata types: `ApiKeyMeta`, `ProviderMeta`, `ProviderKind`, `ProviderCandidate`, `SetupProviderEntry`

### `@vibeframe/core`

- `private: true` — workspace-internal. Exports flow through `@vibeframe/cli`.

### `@vibeframe/mcp-server`

- No library exports; it's a `bin`-only tool published for `claude_desktop_config.json` / `mcp.json` users.

## Historical breaking changes (v0.x)

The CLI command tree saw three meaningful renames during 0.x. All are documented here so a user upgrading from very early versions can update their scripts.

| When | Change | Migration |
|---|---|---|
| **v0.74** | `vibe analyze` / alias `az` → **`vibe inspect`** | Replace `analyze` with `inspect` in scripts. The `analyze` alias was deprecated in v0.74, removed in v0.75. |
| **v0.74** | `vibe pipeline` / alias `pipe` → **`vibe remix`** | Replace `pipeline` with `remix` in scripts. The `pipeline` alias was deprecated in v0.74, removed in v0.75. (Note: `vibe run <yaml>` is the YAML pipeline executor, distinct from the renamed group.) |
| **v0.80** | `vibe project` → **`vibe timeline`** | The `project` alias still works but is deprecated and hidden from `vibe --help` (since v0.88). Will be removed in 1.0. |

Soft-deprecations active in v0.91 (will be removed in 1.0):

- **`vibe edit silence-cut --analyze-only`** — use `vibe detect silence` for read-only detection. The flag prints a stderr deprecation warning since v0.83.
- **`vibe project ...`** — use `vibe timeline ...`. Hidden from main `--help` since v0.88.
- **`vibe generate video -p fal`** — use `-p seedance`. The provider id was renamed in v0.91 (fal.ai is the gateway, Seedance is the model); `-p fal` survives as an alias and prints a stderr deprecation warning when used.

## Drift policies (post-1.0)

To prevent regressions like the `--help` "all 79 commands" / `MODELS.md "(5)"` drift, future docs that cite numeric facts should:

- **Pull from `vibe schema --list` at runtime** (e.g. `vibe schema --list --json | jq length`) rather than hardcoding.
- **Or use build-time injection** like `apps/web/next.config.js` does — counts populated from source so drift is structurally impossible.

The setup wizard, doctor, and `--help` already follow this pattern. Static docs (README / MODELS.md / ROADMAP) are the remaining audit surface.

## What's intentionally **not** ready for 1.0

- **`vibe agent` / `vibe build` / `vibe run` redesign** (Phase B/C from `cli-architecture.md`). The three orchestrators have intentional overlap; consolidating them is a 2.0 conversation. Until then, see `cli-architecture.md` for the boundary statement.
- **fal.ai / Kling / ImgBB live key validation** — these providers don't expose a cheap auth-only endpoint as of 2026-04-30. `vibe doctor --test-keys` reports SKIP with the check date.
- **Plugin system / external providers** — the `defineApiKey` / `defineProvider` registry is currently for first-party use. Public extension API is post-1.0.
